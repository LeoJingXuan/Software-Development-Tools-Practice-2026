实验环境搭建小结

工具链概述

本次实验安装了 Node.js + npm + Git + Hexo 四件套，
它们构成了一个完整的静态博客开发栈。

一个有意思的发现

在 hexo init 的过程中，我注意到命令行输出的依赖安装阶段
实际上就是在执行 npm install。
这意味着 hexo init 本质上是：

  1. git clone 模板仓库
  2. npm install 安装依赖
  3. 生成默认配置文件

理解了这一点，”Hexo 到底是什么”就变得清晰了——
它是一个基于 Node.js 的命令行工具，
借助 npm 包管理能力来组织和加载插件。
